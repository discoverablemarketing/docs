## Base URL

All API requests should be made to:

```
https://chatads--chatads-api-fastapiserver-serve.modal.run
```

## Authentication

All API endpoints require authentication via the `x-api-key` header.

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  https://chatads--chatads-api-fastapiserver-serve.modal.run/v1/chatads/messages
```

Get your API key from [app.getchatads.com/settings/api-keys](https://app.getchatads.com/settings/api-keys).

## Request Format

All requests should:
- Use `Content-Type: application/json`
- Include the `x-api-key` header
- Send data in JSON format

## Response Times

Our API is designed for low-latency inline processing, with response times optimized for different use cases.

<CardGroup cols={2}>
  <Card title="Standard" icon="gauge">
    **350ms - 600ms**

    Full message analysis with keyword extraction, intent scoring, and affiliate link resolution.
  </Card>
  <Card title="Message Analysis = Fast" icon="bolt">
    **250ms - 400ms**

    Faster, with potential accuracy tradeoff. Set `message_analysis: "fast"` for reduced latency.
  </Card>
</CardGroup>

<Tip>
  Use `skip_message_analysis: true` for the fastest responses when you already have a product keyword. This bypasses extraction entirely and goes straight to link resolution.
</Tip>

## Response Format

All responses follow this structure:

```json
{
  "success": boolean,
  "data": object | null,
  "error": object | null,
  "meta": {
    "request_id": string,
    "country": string,
    "language": string,
    "usage": object
  }
}
```
